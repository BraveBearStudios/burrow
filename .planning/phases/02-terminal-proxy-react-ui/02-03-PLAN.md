## Phase Goal

**As an** operator, **I want to** open a running workspace's live Claude Code terminal in
the browser and type into it, **so that** I can start supervising an agent session without
touching the Proxmox host.

<objective>
The MVP vertical slice: the thinnest end-to-end path that proves a real xterm.js terminal
connects through the Wave-1 WS bridge to a (stub) ttyd and echoes typed input — plus the
`useWorkspaces` poll the sidebar/status bar will share. This is the first deliverable where
a user can DO something they could not before: see and type into a live terminal.

Per MVP mode, Task 1 is a failing end-to-end test of the happy path; Task 2 is the thinnest
UI→hook→WS slice that makes it pass (xterm + WS + the verified ttyd frames + a minimal
mounting panel + a one-panel App shell); Task 3 hardens the slice with fit/reflow, jittered
reconnect + overlay, and clean dispose (TERM-05/06/07) and adds the `useWorkspaces` poll.

Purpose: de-risk the load-bearing terminal correctness early and give Waves 3-4 a working
panel + a shared workspace-list hook to build the tiling and surrounding surfaces on.
Output: `useTerminal`, `TerminalPanel`, `useWorkspaces`, a one-panel `App` shell, and the
unit/integration tests proving render/echo/fit/reconnect/dispose.
</objective>
