## Phase Goal

**As an** operator, **I want to** see all my workspaces with live status, spin up a new one
from a repo and watch it boot, and read fleet counts + capacity at a glance, **so that** I
can manage many sessions from the browser without the Proxmox host.

<objective>
The surrounding UI surfaces that complete the operator workflow on top of the Wave-3 tiling
layout: the live-polled `WorkspaceList` sidebar (UI-01), the `Navbar` with per-node capacity
chips, the `NewWorkspaceModal` create flow with boot-progress (UI-03), and the `StatusBar`
counts/uptime/capacity (UI-04). Plus a `useNodes` hook polling the Wave-1 `GET /api/v1/nodes`.

Per MVP mode each task adds a capability a user did not have: Task 1 — see + click-to-focus
all workspaces with live status (sidebar/navbar); Task 2 — create a workspace and watch it
boot into a panel (modal); Task 3 — read fleet health + capacity (status bar) and the full
shell assembled. Every component is graded by gsd-ui-auditor against the 15 UI-SPEC criteria,
so implement the binding visual + state contracts exactly (tokens not hex, gold discipline,
green-only action, hairlines, weight ≤500, self-hosted fonts, a11y).
Output: `WorkspaceList`, `Navbar`, `NewWorkspaceModal`, `StatusBar`, `useNodes`, the finished
`App` shell, and MSW-backed tests.
</objective>
