# Phase 2 — UI Design Contract

> Visual and interaction contract for the Burrow tiling terminal UI (navbar + workspace
> sidebar + react-mosaic terminal grid + status bar). Generated by gsd-ui-researcher,
> verified by gsd-ui-checker, graded by gsd-ui-auditor.
>
> **Binding source:** the committed design handoff —
> `docs/design/burrow-ui-mockup.html`, `docs/design/burrow-ui-design-prompt.md`, and
> `design/Burrow-handoff/burrow/project/_ds/.../colors_and_type.css`. Tokens below are
> **lifted from the mockup**, which the design prompt declares the source of truth.
> **Do not invent a different aesthetic.** When the prompt and the generic `_ds` OSS
> tokens disagree (the `_ds` ships neutral/system-font defaults built *to be overridden*),
> the Burrow mockup wins — it is the `_ds` accent-override the README describes.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (hand-built; Tailwind v4 `@theme` CSS-first, no shadcn) |
| Preset | not applicable |
| Component library | none — react-mosaic-component 6.2.0 (tiling), @xterm/xterm 6 (terminal); all other UI is hand-rolled |
| Icon library | **inline outline SVG only**, 1.5px stroke (1.8px active), `currentColor`. NO Tabler icon-webfont, NO icon font, NO CDN. (The mockup's `@tabler/icons-webfont` CDN `<link>` is a prototype shortcut — do NOT ship it; redraw the handful of glyphs used as inline SVG.) |
| Font (display/brand) | **Space Grotesk** 500 — wordmark, modal titles |
| Font (UI/body) | **Inter** 400–500 — all UI text |
| Font (mono) | **JetBrains Mono** 400 — terminals, repo paths, branch chips, all gold stats |
| Font delivery | **Self-hosted only** (`@font-face`, woff2 under `ui/public/fonts/`, `font-display: swap`). NO Google Fonts / `fonts.googleapis.com` / `fonts.gstatic.com` CDN — that violates the security-header / CSP posture (CLAUDE.md, PLAT-05). If a font cannot be self-hosted at build time, fall back to the `_ds` system stacks (`ui-sans-serif, system-ui…` / `ui-monospace, Menlo…`); never a network font dependency. |
| Theme | `data-theme` on the app root. Ship **all four** themes: `dark` (default/hero), `dark-soft`, `medium`, `light`. Four-swatch switcher in the top bar. Default = `dark`. |
| Max font-weight | **500.** Never 600/700 anywhere in the UI. |
| Casing | **Sentence case** everywhere. Uppercase only for 11px overlines (letter-spacing 1.3px). |
| Depth model | **Flat. No gradients, no shadows, no glow.** Depth comes only from the four background levels + hairline borders. |
| Separator | middle-dot `·` (e.g. `burrow · main`, `node1 · 3 running · 11 GB free`). |

## Spacing Scale

4px base. No arbitrary values. (`_ds` `--space-*` tokens.)

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | icon gaps, dot offsets, inline padding |
| sm | 8px | compact element spacing, button gaps |
| md | 16px | default element spacing, bar horizontal padding |
| lg | 24px | section padding |
| xl | 32px | layout gaps |
| 2xl | 48px | major section breaks |
| 3xl | 64px | page-level spacing |

**Exceptions (fixed chrome heights / widths, from the mockup — load-bearing, not arbitrary):**
top bar `52px` · status bar `32px` · sidebar width `228px` · panel header `36px` · modal
width `400px` · input height `36px` · status dot `7px` · avatar `30px` · brand mark `28px`.
Intra-component paddings use `4px`-grid values (e.g. row `10px/11px`, panel body `11px/13px`,
modal body `17px/19px`) as in the mockup; keep them within ±2px of a grid step.

**Radius:** cards/panels/modal `12px` · panel `10px` · buttons/inputs/nav-rows `8px` ·
branch/badge chips `6px` · dots/avatar `9999px`.

**Motion:** `cubic-bezier(0.16,1,0.3,1)`, `120ms` on hover, `scale(0.98)` on press. Spinner:
`0.8s linear infinite`. Cursor blink: `1.1s steps(1) infinite`.

## Component Visual + State Contracts

Each component below is binding. States the auditor will grade are enumerated. Component
files follow tech-spec §4.1: `ui/src/components/`, `hooks/`, `store/`, `api/`, `types/`.

### Top bar — `Navbar` (UI-04 capacity, host chrome) — 52px, `--bg-surf`, hairline bottom

- **Left:** brand mark = `28px` rounded square (`radius 7px`), `background: --accent`, with a
  **gold** hexagon SVG stroke (`#f4bb62`, 1.8px) centered. Beside it: "Burrow" (Space Grotesk
  500, 15px) with a `WORKSPACES` gold overline (9.5px, tracked 2px) under it.
- **Center:** one capacity chip per node — `--bg-panel`, hairline, `radius 8px`, `12px` text.
  Content: status dot (`--ok`) + `node1 · ` + **gold mono** running count + ` running · ` +
  **gold mono** free-GB. Data from the workspace list grouped by `node` + node capacity.
- **Right:** four circular `14px` theme swatches (each shows its own `--bg`), then the primary
  `+ New workspace` button (green `--accent`, text `#bdd4bd` dark / `#e5f0e5` light, inline `+`
  SVG). Hover `filter: brightness(1.12)`; press `scale(.98)`.
- **States:** loading (chips show skeleton/`—` until first poll) · empty (no nodes → hide
  chips, keep brand + button) · the active theme swatch reads as selected (subtle ring).

### Sidebar — `WorkspaceList` (UI-01) — 228px, `--bg-surf`, hairline right

- **Header:** `WORKSPACES` overline (11px, `--text-muted`) + an overflow-menu dot/`⋯` (inline SVG).
- **List rows** (`radius 8px`, `padding 10px/11px`, `margin-bottom 2px`): each row =
  - **status dot** `7px` (color per status; `margin-top 5px` to align with the name),
  - **name** (Inter 500, 13px, `--text`),
  - **`repo · branch`** below in muted mono (`--text-muted`, 11.5px, JetBrains Mono) — from
    `projectRepo`/`projectBranch`,
  - for **non-running** states, an inline **overline** after the name in the matching status
    color: `creating` (`--warn`), `error` (`--err`), `stopped` (`--text-muted`). `running` shows
    no overline (the green dot is enough).
- **Status → color map:** `running --ok` · `creating --warn` · `error --err` · `stopped --text-muted`.
- **Active row:** `--accent-bg` fill **and** a `2px` `--accent-line` rounded left-edge bar
  (`::before`, inset `9px` top/bottom). Active state is **two-way synced** with the focused
  Mosaic panel (clicking either updates both — `layoutStore.activeWorkspaceId`).
- **Hover:** `--bg-hover`. **Click:** focus the workspace's panel (open it in Mosaic if not
  already open, then mark active).
- **Live polling (UI-01):** TanStack Query polls `GET /api/v1/workspaces` (recommend ~3s
  `refetchInterval`). **Live indication:** a `creating` row's dot pulses (reuse the `_ds`
  `@keyframes pulse`, 1.4s) until it resolves to `running`/`error`; rows reconcile on each poll
  (added/removed/status-changed) without a full re-render flash. `destroyed` rows drop out.
- **States to render:**
  - **loading** (first fetch): 3–4 shimmer rows (no hard spinner block).
  - **empty** (zero workspaces): centered muted copy — heading + body + a hint to use
    `+ New workspace` (see Copywriting).
  - **error** (poll failed): a small inline `--err` strip at the list top — "Couldn't reach the
    control plane. Retrying…" — list keeps last-known rows beneath; auto-recovers on next poll.
- **Footer (pinned, border-top):** user row — `30px` avatar (`--accent-bg` fill, `--accent-line`
  initials, hairline), username (`maintainer`), descriptor (`self-hosted`), gear SVG. Static in
  v1 (no auth — LAN-only).

### Terminal grid — `WorkspaceLayout` (react-mosaic, UI-02) — fills main, `padding 11px`, gap 11px

- **Library:** `react-mosaic-component` **6.2.0**. Tree + active id live in Zustand
  `layoutStore` (`mosaicNode`, `activeWorkspaceId`, `openPanel`/`closePanel`/`splitPanel`).
- **Affordances:** drag (grip in header), split H/V (split icon in header → `splitPanel(id,
  'right'|'bottom')`), resize (drag the gutter between panels — react-mosaic native splitter).
  Gutter is a hairline; on hover it shows `--accent-line` at low emphasis. **Suppress
  react-mosaic's default blueprint/`.mosaic-blueprint-theme` chrome** — import no Blueprint
  CSS; style splitters/drop-targets with Burrow tokens (flat, hairline, no blue).
- **Empty state** (no panels open): centered muted illustration-free message in the grid area —
  heading + body + the `+ New workspace` affordance (see Copywriting). The sidebar may still
  list workspaces; clicking one opens its first panel.
- **Persistence + reconcile (UI-02 + UI-05):** persist the Mosaic tree (localStorage via Zustand
  `persist`); on load, **reconcile against the live workspace list** — drop panels whose
  workspace is gone/`destroyed`; keep panels for `running` ones and reconnect their terminals.

### Terminal panel — `TerminalPanel` + `useTerminal` (TERM-05/06/07) — `--bg-surf`, hairline, `radius 10px`

- **Header (36px, `--bg-panel`, hairline bottom):** drag grip SVG (`--text-muted`, `cursor:grab`)
  · workspace **name** (Inter 500, 12.5px) · **branch chip** (mono 11px, `--bg-panel-alt`,
  hairline, `radius 6px`) · **model label in gold** (`claude-opus-4 · rtk`, mono 10.5px) ·
  spacer · icon buttons (`24px`, transparent, `--text-muted`, hover `--bg-hover`+`--text`):
  **split** (layout-columns SVG) · **detach** (plug SVG) · **terminate** (× SVG).
- **Body (`.term`):** real xterm.js mounted on a `div` ref (TERM-05). `padding 11px/13px`,
  JetBrains Mono 12px, `line-height 1.6`, `color --text-sub`, scroll-y. WS to
  `/ws/workspaces/{id}/terminal`. The live block **cursor** = `7px×14px`, `--accent-line`,
  blink `1.1s steps(1)`.
- **Fit behavior (TERM-05):** `@xterm/addon-fit` + a `ResizeObserver` on the body div → `fit()`
  on every container resize and on Mosaic split/drag/resize end; drives the ttyd resize frame
  (`'1'`+JSON) so the TUI reflows (never stuck 80×24). Debounce fit to one frame.
- **Active panel (UI-02 focus):** `1px` `--accent-line` ring (replaces the hairline). All other
  panels keep the `0.5px` hairline. Synced with the sidebar active row.
- **State: `reconnecting` overlay (TERM-06)** — the load-bearing state. Absolute overlay over the
  body only (`inset:0`, `background rgba(26,28,26,.78)` dark / `rgba(240,242,240,.82)` light;
  per-theme), centered column, `gap 9px`:
  - a `22px` **spinner** (2px border, top-arc `--gold`, `0.8s linear`),
  - copy line (`--text-sub`, 12px): **`reconnecting… attempt {n} / 5`** (sentence case, live
    attempt counter),
  - a secondary **`Reattach`** button (`.btn`, hairline) that forces an immediate retry.
  Backoff: exponential, **5 retries, max 30s**, jittered (curve at Claude's discretion per
  CONTEXT). The overlay shows during every backoff wait; the terminal text stays visible behind
  the translucency. **Detach is not terminate** — the session survives a tab close (UI-05).
- **State: `error`** — when ttyd is unreachable / backoff exhausted / the proxy emits a typed
  `{"type":"error","code":"LXC_NOT_READY"}` frame: overlay swaps to a **non-spinning** `--err`
  state — a small error glyph + copy ("Session unavailable. {reason}.") + a `Retry` button. **Stop
  auto-retrying on `error`/`destroyed`** (per CONTEXT/TERM-06).
- **State: `terminated`/closing** — terminate (×) dims the panel (`opacity .4`) pending a
  confirm, then `closePanel(id)` removes it. Unmount is clean (TERM-07): close WS, `term.dispose()`,
  `ResizeObserver.disconnect()`.
- **State: `connecting`** (initial mount, pre-first-byte): a light overlay with the spinner + a
  muted "Connecting…" — distinct from `reconnecting` (no attempt counter).
- **`useTerminal` returns** `{ containerRef, status, reconnectAttempts }` where
  `status ∈ {connecting, open, reconnecting, error, closed}`; the panel renders overlay per
  `status`.

### New Workspace modal — `NewWorkspaceModal` (UI-03) — centered, 400px, `--bg-surf`, `radius 12px`, hairline-mid

- **Backdrop:** `rgba(8,13,8,.55)`, click-outside closes (form state only). Modal traps focus.
- **Header:** Space Grotesk 500 16px title (`New workspace` → `Creating {name}` in the saga
  state) + a `×` close button.
- **Form state** (`padding 17px/19px`):
  - **fields**, each with an 11px uppercase tracked `--lab` label above:
    `Name` · `Git repo` · row of [`Branch` (flex, default `main`) | `Node` (128px, `node1`/`node2`)].
  - **inputs:** `height 36px`, `--bg-panel-alt`, hairline-mid, `radius 8px`, 13px Inter. **Focus:**
    `border-color: --accent-line`, no glow.
  - **footer (right-aligned):** secondary `Cancel` (`.btn`) + primary green `Create` (`.btn.pri`,
    inline `+` SVG). `Create` → `POST /api/v1/workspaces` (`{name, projectRepo, projectBranch,
    node, pluginSet}`), returns `202`.
  - **validation:** required = Name + Git repo + Node; inline `--err` helper text under the
    offending field, and the field border turns `--err`; `Create` disabled until valid. Surface a
    server `error.message` (envelope) as a `--err` strip above the footer (e.g. capacity refusal,
    CAP-01).
- **Boot-progress / saga state** (swaps the form in place, title → `Creating {name}`): a vertical
  **checklist** (mono, 12.5px) animating the **real create saga** (WS-02), each step muted
  `--text-muted` → full `--text` as it completes. Steps + glyphs:
  1. `✓ Reserving VMID {vmid} · writing row` (creating row + reserved VMID *before* clone, SC-2)
  2. `✓ Cloning template ({src} → {vmid})`
  3. `⟳ Starting LXC · waiting for IP` — **active** step shows a `13px` **gold-topped spinner** as
     its bullet
  4. `○ Waiting for Claude (ttyd health)` — pending = hollow `○`
  - mono **footnote** (`--text-muted`, 11.5px): `POST /api/v1/workspaces → 202 · polling status…`
  - **Driving data:** poll `GET /api/v1/workspaces/{id}` after the `202`; map the workspace
    `status` to the step the checklist is on (`creating` → steps 1–4 advancing; the worker's
    progress is inferred from status + elapsed). On `running` → mark all `✓`, brief pause, then
    `layoutStore.openPanel(id)` + close modal. On `error` → see below.
  - **error state:** any step turns to a red `✕` with the failure reason mono-rendered beneath; a
    `Close` button replaces the progress; the workspace shows `error` in the sidebar (WS-03
    compensation already freed the VMID server-side — the modal just reports it).
- **States to render:** form (default) · validating (inline errors) · submitting (Create →
  spinner) · saga (4 animated steps) · success (auto-close + panel open) · saga-error.

## Responsive Behavior (LAN, any device — browser-first, no native app)

- **Desktop (≥1024px):** full shell as mocked — top bar + 228px sidebar + Mosaic grid + status bar.
- **Tablet (640–1023px):** sidebar collapses to an off-canvas drawer toggled by a hamburger in the
  top bar; Mosaic remains but defaults new opens to a single column; capacity chips may wrap.
- **Phone (<640px):** single-panel focus mode — show the active terminal full-width; sidebar +
  panel switcher behind the drawer; Mosaic splitting disabled (one panel at a time); top/status
  bars stay pinned and never grow. xterm `fit()` runs on orientation change.
- All columns `overflow: hidden` with **inner scroll**; top bar (52px) and status bar (32px)
  **never grow or shrink**. Terminal text never reflows the chrome — only the xterm body scrolls.

## Accessibility Basics

- **Focus:** every interactive element (buttons, theme swatches, rows, inputs, panel icon
  buttons, modal controls) is keyboard-focusable with a **visible focus ring** = `--accent-line`
  (use `:focus-visible`; the only ring color, matching the active-panel ring). Never remove
  outlines without a replacement.
- **Keyboard:** modal traps focus, `Esc` closes the form state, `Enter` submits when valid.
  Sidebar rows are reachable via Tab and activate on `Enter`/`Space`. Mosaic panels are
  focusable; the terminal takes keyboard input when focused (xterm handles it).
- **Contrast:** body text ≥ 4.5:1 on its surface; the gold-on-surface stats and status overlines
  must clear 4.5:1 in every theme (verify `--gold`/`--warn` on `--bg-surf`, especially `light`
  and `medium`). Status is **never color-only** — pair each status dot with a text label
  (`running`/`creating`/`error`/`stopped`) so it survives color-blindness.
- **ARIA/semantics:** app root `aria-label="Burrow workspace manager"`; sidebar = `nav`/`list`
  with `aria-current` on the active row; status dots `aria-hidden` (the adjacent text is the
  accessible name); the reconnecting overlay uses `role="status"` `aria-live="polite"` so the
  attempt counter is announced; the error overlay uses `aria-live="assertive"`. Theme swatches
  carry `aria-label` (already in the mockup) and `aria-pressed` on the active one.
- **Motion:** honor `prefers-reduced-motion` — disable the cursor blink, spinner spin, and pulse
  (show a static state) for users who request it.

## Security / Posture Reconciliation (binding)

The mockup is a prototype; two of its conveniences **must not ship** because they violate the
LAN-only + security-headers + CSP posture (CLAUDE.md, PLAT-05):

1. **No external font CDN.** Drop the `fonts.googleapis.com` / `fonts.gstatic.com`
   `<link preconnect>`. **Self-host** Space Grotesk / Inter / JetBrains Mono as woff2
   (`font-display: swap`), or fall back to the `_ds` system stacks. A strict CSP with no
   `font-src`/`style-src` to Google must pass.
2. **No icon-font CDN.** Drop `@tabler/icons-webfont` from jsdelivr. Use **inline outline SVG**
   (1.5px stroke, `currentColor`) for the ~10 glyphs actually used (hex mark, plus, grip,
   layout-columns, plug, ×, dots/`⋯`, settings, bolt, clock, status check/circle). This matches
   the `_ds` iconography rule ("no icon font, no library dependency").

The xterm WS connects only to the same-origin control plane (`/ws/workspaces/{id}/terminal`);
no third-party network calls from the UI.

## Acceptance-Checkable Visual Criteria (gsd-ui-auditor grades against these)

A reviewer can verify each of these against the running UI without ambiguity:

1. **Tokens, not hex.** No hard-coded colors in components; every color resolves from a
   `--token`. Switching `data-theme` swaps all four themes with no orphaned hardcoded hue.
2. **Gold discipline.** Gold appears **only** on: model labels, stats (tokens-saved, uptime,
   capacity numbers), status-bar counts, the `WORKSPACES` overline, the spinner top-arc. Zero
   gold buttons, gold body text, or gold full-borders. Status dots are `--ok/--warn/--err/--text-muted`,
   never gold.
3. **Green is the only action color.** New-workspace + Create buttons, active-row fill + left
   bar, active-panel 1px ring, input focus, cursor — all `--accent`/`--accent-line`. No second
   interactive hue.
4. **Hairlines + flat.** All borders 0.5px **except** the 1px active-panel ring. No gradients,
   no box-shadows, no glow anywhere.
5. **Weight ≤ 500.** Nothing renders at 600/700. Sentence case everywhere except 11px tracked
   overlines.
6. **Self-hosted fonts only.** Network panel shows **no** request to `fonts.googleapis.com`,
   `fonts.gstatic.com`, or `cdn.jsdelivr.net`. The three Burrow faces load from same-origin
   (or the system fallback renders).
7. **Status mapping.** A `running` row = green dot, no overline; `creating` = amber dot +
   pulsing + `creating` overline; `error` = red dot + `error` overline; `stopped` = muted dot +
   `stopped` overline. Counts in the status bar match the list.
8. **Sidebar ↔ panel sync.** Clicking a row focuses its panel (green ring) and vice-versa; the
   active row shows the `--accent-bg` fill + 2px left bar.
9. **Reconnecting overlay.** Killing a terminal's WS shows the translucent overlay with spinner +
   `reconnecting… attempt n / 5` (counter increments) + `Reattach`; terminal text stays visible
   behind it; it stops at 5 / on `error` and swaps to the error overlay.
10. **Mosaic affordances.** Panels can be opened, split H/V, drag-reordered, and resized; the
    splitter is a Burrow-styled hairline (no Blueprint blue); empty grid shows the
    `No open terminals` state.
11. **Modal saga.** Create runs the 4-step checklist (`✓/✓/⟳/○`) with a gold-topped spinner on the
    active step and the `→ 202 · polling status…` footnote; on `running` the panel opens and the
    modal closes; a failure shows a red `✕` step + reason.
12. **Terminate is confirmed + distinct from detach.** × asks `Destroy {name}? …` before removing
    the panel; the plug (detach) never destroys and is labeled as session-preserving.
13. **Fit/reflow.** Resizing a panel reflows the terminal (not stuck 80×24); unmounting a panel
    leaves no leaked WS/observer (TERM-07).
14. **A11y.** Visible `--accent-line` focus ring on every control; the reconnecting overlay is an
    `aria-live` region; status is never color-only (dot + text label); `prefers-reduced-motion`
    stills the blink/spin/pulse.
15. **Chrome invariants.** Top bar 52px and status bar 32px never grow; sidebar 228px; only inner
    columns scroll.
