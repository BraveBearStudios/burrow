<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → WS proxy (`/ws/workspaces/{id}/terminal`) | untrusted client opens a WS upgrade; only the workspace id (path) is client-supplied |
| WS proxy → worker ttyd (`ws://{lxc_ip}:7681/ws`) | control-plane → LAN worker; the upstream host is derived ONLY from the DB row |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering/Info | terminal.py upstream dial | mitigate | Build `ttyd_url` ONLY from the looked-up `workspace.lxc_ip`; never a query param/header/sub-path (SSRF). Test: only the row's IP is reachable. |
| T-02-02 | Elevation/DoS | terminal.py pre-accept gate | mitigate | `getWorkspace` + `status=="running"` + `lxc_ip` check BEFORE `accept`; else `close(1008)`. Test: `test_rejects_non_running`. |
| T-02-03 | Spoofing (CSWSH) | terminal.py WS upgrade | mitigate | Explicit `Origin`-header check vs `settings.allowed_origin` (Starlette WS does not honor CORS); `close(1008)` on mismatch. |
| T-02-04 | DoS (half-open leak) | terminal.py teardown | mitigate | `FIRST_COMPLETED` + cancel + `ping_interval` keepalive bounds FD growth. Test: `test_teardown_no_halfopen`. |
| T-02-05 | Info disclosure | terminal.py logging | mitigate | `terminal.connected/disconnected` events carry `{}` only; the relay never parses/logs frame content or `lxc_ip` (opaque). |
| T-02-06 | Tampering | nodes.py / getNodeMemory | accept | Read-only capacity numbers, no PII, LAN-only no-auth by design (A6); degrade-not-500 prevents an error oracle. |
</threat_model>
