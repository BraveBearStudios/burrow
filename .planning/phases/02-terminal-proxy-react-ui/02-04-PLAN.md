## Phase Goal

**As an** operator, **I want to** tile several workspace terminals side by side and have my
layout survive a page refresh, **so that** I can supervise multiple agent sessions at once
without rebuilding my view every time.

<objective>
The tiling layer: a Zustand `layoutStore` owning the `MosaicNode` tree + active workspace,
a `WorkspaceLayout` binding `react-mosaic-component@6.2.0` to that store (open/split/drag/
resize, Burrow-styled splitters â€” no Blueprint blue), and persistence + reconcile so the
layout survives a refresh and drops panels for gone workspaces while reconnecting the live
ones (UI-02 + UI-05).

Per MVP mode this extends the Wave-2 single-panel slice into multi-panel tiling: after this
plan a user can do something new â€” open/split/drag/resize several terminals and keep that
arrangement across a refresh. The App shell swaps its single `TerminalPanel` for the
`<Mosaic>`-driven layout.
Output: `layoutStore`, `WorkspaceLayout`, the updated `App` shell, and tests proving the
tree mutations, persistence/reconcile, and restore-after-refresh.
</objective>
