## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| upstream GitHub Action → this repo's runner | A third-party action's code executes in CI with the job's token; a repointed mutable tag injects attacker code |
| release-please job token → repo contents/PRs | The job is elevated to write contents + PRs; an over-broad token could write beyond the release PR |
| GitHub-side release state ↔ workflow config | The bootstrap-sha + manifest seed steer which history/version the first release PR proposes; wrong values mis-tag a release |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

No `@vN` floating tag. |
