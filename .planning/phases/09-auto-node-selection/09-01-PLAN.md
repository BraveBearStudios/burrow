<objective>
Lay the three dependency-free foundations the auto-select logic (Plan 02) and the
modal (Plan 03) build on:

1. Extend `FakeComputeProvider` with an OPTIONAL `node_fractions: dict[str, float]`
   so the multi-node selection matrix can be proven over the Fake (the failing-first
   dependency for every Plan-02 selection test).
2. Add a `worker_nodes` list to `Settings` (default `[default_node]`, DERIVED from the
   resolved `default_node` so an operator override propagates) — the topology source
   auto-select iterates (LOCKED: no new `ComputeProvider` ABC method).
3. Factor a single `_fits(fraction, threshold)` helper out of `nodes.py._node_capacity`
   into `api/lib/capacity.py`, used by BOTH `/nodes` and (in Plan 02) auto-select so
   the UI's displayed capacity and the saga's selection decision never drift. The
   `/nodes` route starts iterating `worker_nodes` (not just `[default_node]`).

Purpose: these are the seam-clean primitives; without them the Plan-02 selection
tests cannot assert multi-node behavior and the UI cannot show all candidate nodes.
Output: an extended Fake, a `worker_nodes` Settings key, a shared capacity helper,
a multi-node `/nodes` route, and three failing-first test files (two unit, one integration).
</objective>
