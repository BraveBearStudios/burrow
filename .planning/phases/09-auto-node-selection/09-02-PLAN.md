<objective>
Insert capacity-aware auto node selection into the create saga, proven over the
Fake's multi-node `node_fractions` (built in Plan 01):

1. `WorkspaceCreate.node` becomes `Optional[str] = None` — `None`/omitted signals
   auto; an explicit string is the unchanged manual path.
2. A new `selectNode()` service method (reads `settings.worker_nodes`, calls only
   `getNodeMemory`, uses the shared `_fits` comparator) returns the least-loaded
   fitting node, tie-breaking by node name ascending, skipping nodes whose
   `getNodeMemory` raises, and raising the EXISTING `CapacityError` with a
   manual-pick-hint message when no node fits.
3. `selectNode()` runs INSIDE `_create_lock` (select -> existing capacity guard ->
   VMID reserve, one serialized critical section — preserves ADR-0010 atomicity).
   LOCKED: do NOT move selection outside the lock.

Purpose: this is the WSX-01 core — the operator can create without picking a node
and get the least-loaded fitting one, with no overcommit and the seam staying clean.
Output: the Optional model field, the `selectNode` method + saga wiring, an additive
`CapacityError` message option, and the full criterion-5 unit + integration matrix.
</objective>

The selection algorithm (LOCKED shape, from 09-RESEARCH.md):
- candidates = `self.settings.worker_nodes`
- for each candidate: `try: f = await self.compute.getNodeMemory(n) except Exception: continue` (skip unreachable, mirrors /nodes degrade-not-500)
- keep `(f, n)` when `_fits(f, threshold)` is True (`f <= threshold`)
- if none fit: `raise CapacityError(...)` with a manual-pick-hint message
- else `sort(key=lambda fn: (fn[0], fn[1]))` (least fraction, tie -> node name ascending) and return `fitting[0][1]`

Existing create saga (api/services/workspaceService.py):
- `_create_lock = asyncio.Lock()` (~126); `createWorkspace` (~129) opens `async with self._create_lock:` (~143).
- Inside the lock: step-0 guard `if await self.compute.getNodeMemory(payload.node) > self.settings.capacity_threshold: raise CapacityError(payload.node)` (~145), then step-1 `ws = await self._reserve_vmid_and_row(payload)` (~148).
- `_reserve_vmid_and_row` builds `base = {... "node": payload.node ...}` (~206) — the persisted row's `node` must be the CHOSEN node, not None.
- After the lock: clone/start/ip/ttyd/running tail using `payload.node`. The chosen node must flow through here unchanged.

Existing CapacityError (api/lib/errors.py):
- `class CapacityError(ServiceError): code = "capacity_exceeded"; def __init__(self, node: str) -> None: self.node = node; super().__init__(f"node {node!r} is over the capacity threshold")`.
- It currently takes only `node` and formats a FIXED message. The no-fit case needs a
  different operator-facing message (manual-pick hint) while keeping `code` and the
  existing single-arg `CapacityError(node)` callers working.

Shared comparator (from Plan 01): `from lib.capacity import _fits` -> `_fits(fraction, threshold) -> bool` (`fraction <= threshold`).

Settings (from Plan 01): `settings.worker_nodes: list[str]` (default `[default_node]`),
`settings.capacity_threshold: float` (0.80). The service reads `self.settings.*`.

UNIT test construction (api/tests/unit/test_capacity_guard.py — the analog to mirror):

- For `selectNode` UNIT tests no DB/saga is needed: construct the service over a

INTEGRATION test construction (api/tests/integration/ — app-level, IMPORTANT):
