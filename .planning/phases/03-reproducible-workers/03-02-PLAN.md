<!-- The manifest schema shape (RESEARCH §Manifest JSON-Schema — the shared source of truth).
     Object keyed by plugin name; each entry { source, ref, type }. -->
manifest.json:
```
{ "schemaVersion": "<str>",
  "plugins": {
    "<name>": { "source": "<str>", "ref": "<immutable tag/SHA>", "type": "claude-plugin|binary|npm-global" }
  } }
```
- `type` enum drives bake-vs-pull: claude-plugin → pulled fresh at boot; binary/npm-global → baked (skipped at boot).
- `additionalProperties: false` + the `type` enum is what makes "unknown type fails" provable (fail-closed).
- For a claude-plugin a mutable `ref: "main"` is non-reproducible (Pitfall 1) — pin a tag/SHA.

The baked reference plugins (provision-template.sh lines 49-56): `rtk` = binary, `gsd` = npm-global.

REUSE.toml: manifest.json + manifest.schema.json are comment-less JSON → they CANNOT hold an inline SPDX header; add a `[[annotations]]` block (mirror the existing comment-less-JSON block, lines 20-30) — do NOT add a `//` comment a JSON parser would reject.
</interfaces>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| committed `manifest.json` → boot-time install | a manifest entry directs a `git clone` at boot; a malicious/typo'd entry must not run unchecked |
| plugin `source`/`ref` → `git clone --branch <ref>` | an unpinned (mutable) ref breaks reproducibility |
| CI (`test_manifest_schema.py`) → committed manifest | the gate that catches manifest drift before it ships |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | malicious/typo'd manifest entry at boot | mitigate | fail-closed jq gate at boot (required keys + `type` enum) AND the JSON-Schema CI test; an unknown `type` ⇒ non-zero exit, never skip-and-continue (the locked decision). |
| T-03-07 | Tampering / Reproducibility | cloning a mutable (`main`) plugin ref | mitigate | manifest pins each claude-plugin `ref` to a tag/SHA; `install_claude_plugin` does `rm -rf` + `git clone --depth=1 --branch <ref>` so two boots = identical tree (SC-2). The schema/CI test + ADR-0009 (Plan 03) record that `main` is intentionally non-reproducible. |
| T-03-08 | Information Disclosure | manifest committed to the repo | accept | the manifest holds only non-secret `source`/`ref` placeholders/public repos (CLAUDE.md "never commit secrets / no real topology"); no credential is in the manifest — the clone credential comes from the bootconfig endpoint at boot (Plan 01). |
</threat_model>
