<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| control plane → worker ttyd (:7681) | The WS terminal proxy reaches ttyd over the worker's LAN address; ttyd is LAN-exposed, not lo-only |
| control-plane bootconfig endpoint → worker | At boot the worker pulls non-secret config + a short-lived git credential (Phase 3 fills the fetch) |
| short-lived git credential → worker filesystem | The clone credential must be used and discarded, never persisted to the worker env |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-07-SECRET | Information Disclosure | git credential written to /etc/burrow/worker.env | mitigate | Pull-at-boot: the short-lived repo-scoped credential is fetched at boot, used for clone, and DISCARDED — never written to worker.env (SC-4 / Pitfall 13). Phase 0 leaves the fetch a documented stub; the boot script never persists a secret. |
| T-00-07-EXPOSE | Spoofing / Information Disclosure | ttyd bound to LAN widens exposure vs lo | accept (v1 LAN-only) | LAN bind is required for the proxy to reach :7681 (WORK-04 / SC-9); accepted under the v1 LAN-only-no-auth posture (ADR-0007). Never expose worker :7681 beyond the LAN; nginx/control-plane binds the LAN interface only (Pitfall 12). The hosted path adds auth additively. |
| T-00-07-SUPPLY | Tampering / Supply Chain | NodeSource `curl | bash` + npm i -g claude-code in the template | mitigate | Pin `@anthropic-ai/claude-code@2.1.170` and record the NodeSource setup ref; the `curl|bash` executes on the worker (not CI) and is the operator's documented trust decision (RESEARCH Package Legitimacy Audit note). |
| T-00-07-BOOTFAIL | Denial of Service | silent boot hang masks a failed worker | mitigate | `set -euo pipefail` + error trapping so a boot failure surfaces (Phase 3 emits a typed `boot.error`); the systemd unit Restart=on-failure retries rather than hanging. |
</threat_model>
