## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator shell → Proxmox (pveum/pct) | Day-0 scripts run as root@pam; create a least-privilege identity for the control plane |
| Proxmox token value → .env / disk | The single high-value secret; printed once, written to a gitignored file |
| .env on disk → git / logs / process table | The secret must never reach VCS, CI logs, or `ps` output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-06-DISCLOSE | Information Disclosure | Proxmox token in git/logs/`ps` | mitigate | `read -rsp` (silent), `{ set +x; } 2>/dev/null` around the write, `printf` (never `echo`), never token as a CLI arg, `.env` 0600, refuse to write unless `git check-ignore .env` passes; gitleaks backstop (ASVS V6). |
| T-00-06-EOP | Elevation of Privilege | over-broad Proxmox role | mitigate | Exactly the 9-privilege `BurrowProvisioner` role (no Sys.Modify/Permissions.Modify/Pool.Allocate); scoped to `/pool/burrow-workers` + `/storage/<rootfs>` + `/nodes/<node>`; privsep=1 token (effective = user∩token) (ASVS V4). |
| T-00-06-PRIVSEP | Elevation of Privilege | token granted to user but not token (silent 403, or worse a non-privsep token = full user rights) | mitigate | Token created `--privsep 1`; ACL loop grants the role to BOTH `--users` and `--tokens` at every path; STEP-0 gate verifies via `pvesh get /access/permissions --token`. |
| T-00-06-TAMPER | Tampering | MITM on the Proxmox API (self-signed cert) | mitigate (downstream) | Kit records `PROXMOX_CA_CERT_PATH` in .env (pin the node CA); the control plane uses it, NEVER `verify_ssl=False`. |
| T-00-06-IDEMPOTENT | Tampering | second token-add silently churns tokens / orphans the live one | mitigate | Token guarded `list|grep || add`; on a pre-existing token the script PROMPTS rotate-or-reuse rather than silently creating a second token. |
</threat_model>
