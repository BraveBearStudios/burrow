# Phase 18 Plan 02: Credential Store Frontend (Wizard + Settings + Audit)

**Requirements:** CRED-02, CRED-03, CRED-04 (frontend of CRED-05)
**Surface:** `ui/`

## Task 1 — Admin store + header + types + hooks (CRED-02)
- `ui/src/store/useAdminStore.ts` — Zustand, in-memory only (NO persist middleware):
  `{ secret: string | null, setSecret, clear }`.
- `ui/src/types/setup.ts` — add `AdminSecretBody`, `SaveCredentialsBody`,
  `CredentialStatus` (`{proxmoxTokenSet, proxmoxTokenLast4, gitTokenSet,
  gitTokenLast4, credentialsUpdatedAt}` — match the backend `getCredentialStatus`
  shape), `AuditEntry` (`{id, action, target, outcome, sourceIp, detail, createdAt}`),
  `AuditList` (`{entries: AuditEntry[]}`).
- `ui/src/hooks/useSetup.ts` — add: `useSetAdminSecret` (POST `/setup/admin-secret`),
  `useSaveCredentials` (POST `/setup/credentials`, admin header), `useCredentialStatus`
  (GET `/setup/credentials`, admin header, query), `useAudit` (GET `/setup/audit`,
  admin header, query). Admin-gated hooks read the secret from `useAdminStore` and
  pass `{ headers: { "X-Burrow-Admin": secret } }`; on an `admin_auth` ApiError they
  call `clear()`.

## Task 2 — Wizard steps (CRED-03)
- `SetupWizard.tsx`: add an admin-secret step and a credentials step (after health,
  before/at create). Admin-secret sets the store secret + calls `useSetAdminSecret`;
  credentials step submits Proxmox token (required) + GitHub PAT (optional) via
  `useSaveCredentials`. Password inputs, green `--accent`, transient state only,
  match the existing step a11y + auto-advance. Update `SetupWizard.test.tsx` (MSW).

## Task 3 — Credentials/Settings screen + audit panel (CRED-04, CRED-05 UI)
- `ui/src/components/CredentialsScreen.tsx` — admin-gated post-setup screen:
  prompts for the admin secret if the store is empty; shows credential status
  (set + last4 + updatedAt) with a rotation form (re-submit `useSaveCredentials`);
  never renders a secret value. Includes an `AuditPanel` (read-only, mono,
  newest-first) fed by `useAudit`.
- `Navbar.tsx` — add an entry point (a Settings/Credentials link/button) that opens
  the screen. Mount per the existing layout pattern.
- Tests: `CredentialsScreen.test.tsx` (vitest + MSW) — admin prompt, status render,
  rotation submit, audit render, 401-clears-store.

## Verification
`npm run` typecheck (`tsc -b`/`tsc --noEmit`), biome check, and the full vitest suite
green (new hooks + wizard steps + credentials screen). No secret persisted to
localStorage/cache (assert in tests). SPDX on every new file.
