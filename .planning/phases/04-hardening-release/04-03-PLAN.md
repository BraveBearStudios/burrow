<objective>
Build the per-workspace activity drawer (UI-06) that surfaces the workspace event log.

This plan delivers a right-side slide-in drawer opened from the `TerminalPanel` header that polls `GET /api/v1/workspaces/{id}/events` via TanStack Query (only while open), reverses the oldest-first endpoint to newest-first, and renders each event with a color-coded namespaced-type badge + redacted `data` summary, with `boot.error` emphasized. It is hand-built on the existing Phase-2 design tokens (no new runtime UI dependency) with full a11y (role=dialog, focus trap, Esc, focus-return). The binding visual/interaction contract is 04-UI-SPEC.md.

Purpose: Give the operator an audit trail of each workspace's lifecycle (created/started/stopped/destroyed/terminal connect-disconnect/boot.error/reaper.*) directly in the UI, reusing the shipped events endpoint with zero new backend route.
Output: `WorkspaceEvent` type, `useWorkspaceEvents` hook, `EVENT_BADGE` map, `ActivityDrawer` component + the panel trigger, a vitest component test, and a Playwright e2e journey.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser drawer → events endpoint | The drawer's only network call; a same-origin read-only poll |
| event `data` → rendered DOM | The drawer renders server-redacted `data`; it must never re-expand/un-redact or re-fetch raw values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03A | Information Disclosure | rendered event `data` | mitigate | The `data` is already server-redacted via `_safe()` before the event is written (verified in workspaceService.py); the UI renders it verbatim and makes ONLY the same-origin events poll — no second request, no un-redact (UI-SPEC §Security, criterion 12) |
| T-04-03B | Tampering / Injection | unknown event type rendering | mitigate | Unknown/forward-compatible types render their raw string in mono via the fallback (no eval, no dangerouslySetInnerHTML); React escapes the text so a crafted type string cannot inject markup |
| T-04-03C | Spoofing | external font/icon CDN | mitigate | Inline outline SVG only, self-hosted fonts inherited from the Phase-2 shell; no third-party request (PLAT-05) |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | no new runtime UI dependency (UI-SPEC Registry Safety: the drawer is hand-built on existing tokens); nothing to gate |
</threat_model>
