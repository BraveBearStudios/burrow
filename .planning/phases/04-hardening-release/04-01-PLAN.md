<objective>
Build the fleet reconciler — the reaper and idle auto-stop — and wire it into the FastAPI lifespan.

This plan adds `Reconciler.reconcile_once()` (a pure single pass over the two provider seams) that (a) destroys pool CTs with no live DB row, frees leaked VMIDs, and marks timed-out `creating` rows `error` while destroying their CTs, under a load-bearing pool-range safety bound; and (b) auto-stops running workspaces idle past the window, derived from the terminal connect/disconnect event log, calling the guarded `stopWorkspace(reason="idle")`. A FastAPI `lifespan` (none exists today) owns a thin periodic loop that calls `reconcile_once()` and cancels cleanly on shutdown.

Purpose: Keep the fleet healthy unattended (CAP-03 reaper, CAP-02 auto-stop) as the crash safety net beyond per-step saga compensation (SC-11), with every decision a pure function of (DB state, compute state, injected `now`) so it is fully CI-provable over the Fake with no real clock or Proxmox.
Output: `reconciler.py`, a `lifespan` in `main.py`, a single-pass unit test suite, and a lifespan start/cancel test.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01A | Tampering / DoS | reaper orphan-destroy | mitigate | Pool-range safety bound: destroy ONLY a vmid that is BOTH in `[worker_pool_start, worker_pool_end]` AND absent from live DB rows; the Fake's unfiltered usedVmids forces the `if vmid in pool` re-assert in the reconciler (V4 authorization-of-action); a test asserts an out-of-pool CT is untouched |
| T-04-01B | DoS | idle auto-stop | mitigate | Idle requires the LAST terminal event to be a `terminal.disconnected` older than the window; a reconnect appends `terminal.connected` and re-arms (Pitfall 2); a connect->disconnect->connect test asserts no stop |
| T-04-01C | Information Disclosure | `reaper.*` events + structured logs | mitigate | All reaper event/log text routed through the existing `_safe()` redactor (git/CI tokens, URL userinfo, long opaque tokens, length cap, V7); row-less orphan reaps go to structured logs (events FK needs a live workspaceId), redacted |
| T-04-01D | Tampering | reaper reaping an in-flight create | mitigate | `live_vmids` includes `creating` rows (persisted before clone, SC-2) so an in-flight saga's vmid is not an orphan; the timed-out sweep uses a generous `creating_timeout_s` well beyond a normal create (Pitfall 1); a fresh-creating-not-swept test guards it |
| T-04-01E | Elevation / Tampering | seam leakage (new Proxmox/SQLite call) | mitigate | reconciler imports only the two ABCs + Settings + `_safe`; `test_seam_leakage.py` tokenize guard fails the build on any `proxmoxer`/`aiosqlite` code token; no `verify_ssl=False` regression (no new Proxmox call) |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | no new runtime dependency (RESEARCH Standard Stack: NONE); injected-`now` replaces freezegun; nothing to gate |
