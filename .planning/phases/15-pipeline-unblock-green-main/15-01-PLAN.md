<objective>
Fix RELX-04 and RELX-06 in `.github/workflows/release.yml`: build every GHCR image reference
from a single lowercased owner (the mixed-case `BraveBearStudios` currently makes syft
case-mismatch the pushed image, so the SBOM step fails and cosign sign + SLSA attest never run,
shipping unsigned partial-publish images), give the two syft SBOM steps registry auth so they can
pull the pushed-by-digest image, and narrow the tag trigger to semver so hand-pushed milestone
tags stop firing the publish path.

Purpose: this is the signed-and-attested half of the "green pipeline" gate. `docker/metadata-action`
already lowercases the PUSHED tag, so the image lands at `ghcr.io/bravebearstudios/...`; the
SBOM/sign/attest steps build their own ref from the raw `github.repository_owner` and desync from
the pushed digest. One lowercased base output, reused everywhere, plus explicit syft auth, fixes the
whole chain uniformly.

Output: edited `release.yml` (lowercase base step + reuse across metadata/SBOM/sign/attest + syft
auth env + semver tag glob) and a lowercase-owner clarification in the `14-ACCEPTANCE.md` verify
runbook. The signed+attested GREEN proof is a live tagged release run (Phase 20); this plan's proof
is structural: parseable YAML, no raw-owner GHCR ref, syft auth present, SHA-pins intact.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub-hosted runner -> GHCR registry | The publish job pushes the image and syft pulls it back by digest for the SBOM; a ref/case desync silently breaks the pull. |
| GitHub-hosted runner -> Sigstore / OIDC | cosign keyless sign + SLSA attest bind to the pushed digest via the run's OIDC identity. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01 | Tampering | GHCR image reference case across SBOM/sign/attest | mitigate | One lowercased `steps.imgref.outputs.base` is reused for metadata, SBOM, cosign sign, and attest subject, so the signed/attested subject is exactly the pushed digest ref (the mixed-case ref previously desynced syft from the pushed image, yielding unsigned publishes). |
| T-15-02 | Information Disclosure | GITHUB_TOKEN passed to syft as SYFT_REGISTRY_AUTH_PASSWORD | accept | Scoped to the run and to the ghcr.io authority only; it is the same GITHUB_TOKEN already used by docker/login-action, no new long-lived secret introduced. |
| T-15-03 | Tampering | release publish trigger glob | mitigate | Narrowed to semver `v[0-9]+.[0-9]+.[0-9]+` so an arbitrary hand-pushed `v*` tag can no longer fire the sign/attest/publish path; release-please owns release tags. |
| T-15-SC | Tampering | npm/pip/cargo installs | mitigate | No package-manager installs in this plan (workflow YAML + doc edits only) and no new `uses:` action; no Package Legitimacy Audit required. |
</threat_model>
