<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Base image registry (Docker Hub / GHCR) -> build | Base image contents enter the build from an upstream registry; a floating tag would let contents drift between builds. |
| Trivy scanner -> gate policy | The gate decides pass/fail on scanned CVEs; ignore rules and suppressions weaken it if unaudited. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | Tampering | base image digest repin | mitigate | Each base stays DIGEST-pinned (sha256), only repinned to the current index digest for the SAME tag; digests are resolved from the registry at edit time, never a floating tag, so reproducibility holds. |
| T-15-05 | Tampering | ignore-unfixed + .trivyignore on the gate | mitigate | ignore-unfixed:true drops only CVEs with no available fix; fixable HIGH/CRITICAL still fail the gate and are cleared by a base repin (never allowlisted); the `if: always()` SARIF run keeps ALL findings for code scanning, so nothing is hidden from the security dashboard. |
| T-15-06 | Repudiation | unreviewed CVE suppression | mitigate | The `.trivyignore` per-entry format forces owner + reviewed date + advisory link and holds only residual UNFIXABLE CVEs, so every suppression is attributable, dated, and justified as unfixable; blanket un-annotated mutes and fixable-CVE allowlisting are disallowed by the documented policy. |
| T-15-SC | Tampering | npm/pip/cargo installs | mitigate | No package-manager installs in this plan (a workflow input change, base-image digest repins, and one policy file); no new `uses:` action; no Package Legitimacy Audit required. |
</threat_model>
