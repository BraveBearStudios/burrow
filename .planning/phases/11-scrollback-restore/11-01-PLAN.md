<objective>
Wrap the worker shell that ttyd execs in `tmux new-session -A -s burrow` so a
web-client / ttyd reconnect reattaches to the still-running worker's tmux session
and sees prior scrollback (criterion 1), prove the `-A` reattach is idempotent
across two boots hermetically (criterion 3), and author ADR-0014 recording the
honest contract.

Purpose: WSX-03 scrollback restore - deliver the tmux `-A` REATTACH MECHANISM
(scrollback survives a ttyd/web-client reconnect to a STILL-RUNNING worker),
proven via the hermetic stub-ttyd argv harness.

Output:
- `burrow-boot.sh` ttyd exec line wraps the shell in `tmux new-session -A -s burrow`.
- `test_burrow_boot.py` argv assertion + second-boot reattach test (both hermetic).
- `docs/adr/ADR-0014-tmux-scrollback.md`.

CRITICAL - do NOT over-claim: this delivers reattach-on-RECONNECT to a running
worker. It does NOT make scrollback survive a real `pct stop` (the tmux server
lives in the LXC's memory and dies on CT halt). Cross-reboot scrollback
persistence (pipe-pane / CRIU) is WSX-06, deferred to v1.4. ADR-0014, the plan,
and every acceptance string MUST state this honestly.
</objective>
