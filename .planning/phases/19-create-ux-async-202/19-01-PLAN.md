# Phase 19 Plan 01: Create-UX Async-202

**Requirement:** UX-01 · **ADR:** ADR-0017

## Task 1 — Saga split (workspaceService.py)
- Extract `reserveWorkspace(self, payload) -> Workspace`: the `async with
  self._create_lock:` block (node resolve + capacity guard + `_reserve_vmid_and_row`).
- Extract `_runBootSaga(self, ws, payload) -> Workspace`: steps 2-7 + the
  compensation `except` (node = `ws.node`; `vmid = ws.vmid`).
- Rewrite `createWorkspace` = `ws = await reserveWorkspace(payload); return await
  _runBootSaga(ws, payload)` — behavior unchanged (existing tests stay green).
- Add `scheduleCreate(self, payload, *, schedule) -> Workspace`: `ws = await
  reserveWorkspace(payload)`; `schedule(self._run_boot_saga_safe(ws, payload))`;
  return `ws`. `_run_boot_saga_safe` awaits `_runBootSaga` in try/except that logs +
  swallows (row already lands `error`). `schedule` is a callback injected by main.py
  (keeps the service free of app-registry coupling).

## Task 2 — App-scoped task registry + lifespan (main.py)
- Module-level `_create_tasks: set[asyncio.Task[Any]]` + `schedule_create_task(coro)`
  that does `t = asyncio.create_task(coro); _create_tasks.add(t);
  t.add_done_callback(_create_tasks.discard)`.
- In `lifespan` shutdown: after cancelling the reconciler, cancel each task in
  `_create_tasks` and `await asyncio.gather(*_create_tasks, return_exceptions=True)`
  (suppress `CancelledError`) so no create task leaks past shutdown.

## Task 3 — 202 endpoint (routers/workspaces.py)
- `create_workspace` → `ws = await service.scheduleCreate(payload,
  schedule=schedule_create_task)`; return the envelope with `status_code=202` (set a
  `Response`/`JSONResponse` status or the route `status_code=202`). Docstring notes
  the async-202 contract (creating row now, poll drives running/error).

## Task 4 — ADR-0017
- `docs/adr/ADR-0017-async-202-create.md` (ADR-0013/0014/0015 format, SPDX, no em
  dashes): context (the ~60s real boot 504), decision (202 + tracked background
  saga, `creating` reused, list-poll driven), consequences (task lifecycle, shutdown
  cancel, at-least-the-row-lands-error guarantee), alternatives (BackgroundTasks,
  client long-poll) rejected.

## Task 5 — UI wait-state
- `NewWorkspaceModal.tsx` + the `SetupWizard.tsx` create step: on a 202 (create
  mutation resolves) close the modal / advance the wizard immediately; rely on the
  existing 3s list `refetchInterval` to show `creating`→`running`/`error`. No new
  poll. Update the affected vitest tests.

## Verification

green (modal/wizard close-on-202). SPDX on ADR-0017; reuse lint clean.
