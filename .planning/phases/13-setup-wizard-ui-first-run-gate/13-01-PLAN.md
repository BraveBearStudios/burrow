## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Tampering | POST /setup/complete (no auth, v1 LAN-only) | accept | v1 LAN-only no-auth is the documented posture (CLAUDE.md security posture); the setter only writes a timestamp — no privilege escalation, no destructive effect. Re-stamping is harmless. |
| T-13-02 | Information disclosure | /setup/state + /setup/complete responses | mitigate | Neither endpoint accepts or returns any secret — `setupCompletedAt` is a timestamp only. No token, no host, no PVE credential touches these routes (the token path is Phase 12 test-connection, redacted). |
| T-13-03 | Denial of service | repeated POST /setup/complete | accept | Idempotent single-row UPDATE WHERE id=1; bounded cost, no row growth, no resource allocation. LAN-only attack surface. |
| T-13-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs in this plan — backend uses only existing deps (fastapi, aiosqlite, pydantic). No legitimacy checkpoint required. |

No unmitigated HIGH. No new secret-at-rest (the setter writes only a timestamp; the operator token never reaches these routes).
