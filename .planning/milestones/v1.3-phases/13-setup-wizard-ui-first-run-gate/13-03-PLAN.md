<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator browser → step-1 token field | the Proxmox token is typed into a transient password field, sent to the Phase 12 redacted /setup/test-connection |
| React component state → any persisted store | the token must never cross into Zustand/localStorage/query cache |
| the gate (UX) → the app | v1 LAN-only no-auth: the gate is a UX gate, not an authz boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-07 | Information disclosure | step-1 token field | mitigate | The token lives ONLY in the step-1 form state (useState), passed to useTestConnection.mutate. It is NEVER written to the query cache, Zustand, or localStorage, and NEVER logged (no console.* of the token/body). The token field is type="password"; the helper copy states it is validated in memory only. |
| T-13-08 | Spoofing / elevation | the first-run gate | accept | The gate is a UX gate only (v1 LAN-only no-auth, CLAUDE.md). It blocks the UI until configured but is not an authz boundary; bypassing it grants no privilege beyond the already-open LAN API. |
| T-13-09 | Tampering | error message rendering | mitigate | The wizard surfaces only the FIXED token-free backend messages (mapped by ApiError.code from main.py _SAFE_ERROR_MESSAGES) plus static UI guidance — it never interpolates the token or raw exception text into the DOM. |
| T-13-SC | Tampering | npm installs | mitigate | No new npm packages — uses existing @tanstack/react-query, react, MSW, vitest, @testing-library. No legitimacy checkpoint required. |

No unmitigated HIGH. Token is transient form state, never persisted/logged client-side; only fixed token-free messages reach the DOM.
