<objective>
Bake tmux into the golden worker template (Ubuntu 24.04 apt) and write a minimal
`/etc/tmux.conf` at provision time so every clone has tmux 3.4 plus the two
scrollback settings the reattach mechanism depends on (criterion 2).

Purpose: WSX-03 - the worker-side software baseline for scrollback restore. Without
the baked tmux + `/etc/tmux.conf`, the `tmux new-session -A -s burrow` wrap from
Plan 11-01 has no tmux to exec and no history/resize config.

Output:
- `tmux` added to the existing apt-install line (unpinned in the line, like ttyd/jq).
- A tmux 3.4 (Ubuntu 24.04) version note in the top-of-file pin comment.
- A baked `/etc/tmux.conf` drop-file containing exactly `set -g history-limit 50000`
  and `set -g window-size latest` - nothing else (YAGNI: no mouse/status/theme).

This plan is independent of Plan 11-01 (different file, no overlap) - both run in
Wave 1 in parallel.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| apt registry → template CT | tmux is pulled from the Ubuntu 24.04 apt baseline at provision time (one-time, inside the template build) |
| baked /etc/tmux.conf → every clone | a root-owned system config baked once, read by tmux on each worker boot; no runtime ingress |

This phase adds NO new network ingress, NO auth surface, and NO secret handling.
The tmux package and `/etc/tmux.conf` are baked once inside the template CT.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-02 | Tampering | tmux from Ubuntu 24.04 apt | accept | distro-packaged tmux 3.4 from the already-trusted, pinned 24.04 apt baseline (same trust decision as ttyd/jq/build-essential); no new third-party/PyPI/npm supply-chain ingress, no apt source added |
| T-11-03 | Denial of Service | unbounded scrollback memory in the tmux server | mitigate | `/etc/tmux.conf` caps per-pane scrollback at `history-limit 50000` (~few MB/pane), bounded rather than unlimited |
| T-11-04 | Tampering | baked /etc/tmux.conf integrity | accept | root-owned 0644 system config baked once inside the template CT (mode parity with worker.env / the systemd unit); the LXC boundary is the control, consistent with the existing template artifacts |

No HIGH-severity threat. tmux is a distro package on the existing apt trust path;
the only behavioral knob is a bounded scrollback buffer. No relay or control-plane
change (criterion 4 is upheld trivially - this plan touches only the worker template).
