ui HEALTHCHECK: nginx 200 on / (wget if present in alpine, else a small nginx-available probe).
Pitfall 8: COPY --chown to the non-root user so files are readable after USER switch.

api build: uv sync --frozen (working dir api/; uv.lock committed). ui build: npm ci then vite build; serve dist/ via nginx.

ci.yml conventions (.github/workflows/ci.yml lines 9-10,19-21,26-27,30-36):

  run #1 aquasecurity/trivy-action format:table severity:'HIGH,CRITICAL' exit-code:'1' ignore-unfixed:'false' (the GATE);
  run #2 if:always() format:sarif output:trivy.sarif (the full REPORT) -> github/codeql-action/upload-sarif if:always().
  ONE run cannot do both (SARIF ignores the severity-scoped exit-code, Pitfall 6 / trivy-action#309). Run for BOTH images (matrix).
  The build-scan job needs `permissions: contents: read, security-events: write` (upload-sarif).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build context → image layer | `.git`/`.env*`/secrets must not leak into an image layer |
| container runtime → host | A root container widens the escape blast radius |
| third-party GitHub Action → CI runner | A compromised/typosquatted action runs in CI |
| built image → registry/deploy | An unscanned image with HIGH/CRITICAL CVEs must not pass the gate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04A | Information Disclosure | Docker build context | mitigate | `.dockerignore` excludes `.git`, `.env*`, tests, local state so no secret enters a layer (ci-cd §2.2) |
| T-04-04B | Elevation of Privilege | container runtime user | mitigate | Non-root `USER`, `COPY --chown`, read-only root FS where possible, drop unneeded caps (ci-cd §2.2 / Pitfall 8) |
| T-04-04C | Tampering / Elevation | third-party actions in build-scan | mitigate | Every new action (setup-buildx, build-push, trivy-action, upload-sarif) SHA-pinned with a version comment; job `permissions` are exactly `contents: read` + `security-events: write` (Pitfall 14) |
| T-04-04D | Spoofing / Tampering | vulnerable image reaching publish | mitigate | Fail-closed Trivy gate (HIGH/CRITICAL exit-code 1) BEFORE any publish; the SARIF run surfaces the full report to code scanning (Pitfall 6) |
| T-04-04E | Information Disclosure | build job pushing on PR | mitigate | `push: false` on PR — the build only proves it builds; publish is release.yml only (ci-cd §2.4); v1 no public exposure (Pitfall 12) |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | no new runtime dependency; the CI additions are SHA-pinned GitHub Actions in YAML, governed by the existing pin convention — nothing to slopcheck |
</threat_model>
