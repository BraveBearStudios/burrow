## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-12 | Tampering | request bodies / status filter | mitigate | Pydantic WorkspaceCreate validation; status is a query param consumed only by parameterized SQL in the provider (ASVS V5) |
| T-01-13 | Tampering | CORS / missing headers | mitigate | Non-* CORS from Settings.allowed_origin (incompatible-with-credentials wildcard rejected); SecurityHeadersMiddleware on every response (PLAT-05, Pitfall 12) |
| T-01-14 | Information Disclosure | error messages / logs | mitigate | Typed errors → safe code+message envelope (no internals); JsonFormatter whitelists fields, no secret extra (ASVS V7) |
| T-01-15 | Information Disclosure | /health leaking internals | mitigate | /health returns only ok/error per dependency; no exception text or connection string in the body |
| T-01-16 | Spoofing | v1 has no auth (LAN-only by design) | accept | v1 is LAN-only no-auth (tech-spec §13, CLAUDE.md); the LAN boundary is the trust precondition — do NOT retrofit auth into v1 paths |
| T-01-SC | Tampering | package installs | mitigate | No new packages (stdlib logging per A1); `uv lock --check` CI gate |
</threat_model>
