<objective>
Build the pull-at-boot bootconfig endpoint — the one real ASVS L1 surface of this phase. `GET /api/v1/internal/bootconfig/{vmid}` returns a worker's non-secret boot identifiers plus a short-lived, repo-scoped git credential minted per-fetch and discarded, with a vmid-in-pool gate, enumeration-resistant 404s, optional source-IP defense-in-depth, and a hard rule that the credential never reaches a log line or event payload.

Purpose: WORK-03 / ADR-0002 — the worker fetches its config at boot because `pct exec`/`pct push` are absent from the HTTPS API. The endpoint contract is built here; the `burrow-boot.sh` consumer pull-step is Phase 3. The credential issuer is a pluggable seam (A3) — v1 must NOT hard-code a long-lived PAT.
Output: `routers/internal.py`, a `get_by_vmid` lookup + `mint_repo_credential` seam on the service, `main.py` registration, an `IllegalVmidError`, and an integration test proving the in-pool/out-of-pool/no-workspace paths and the no-credential-in-logs guarantee.

Depends on Plan 03 (service) and Plan 04 (routers + middleware + main.py registration pattern + integration conftest). This is the last wave because it touches both `services/workspaceService.py` (Plan 03) and `main.py` (Plan 04). The `git_credential_token` and `bootconfig_source_ip_check` Settings keys were already added by Plan 01 (single-owner config); this plan READS them and does NOT edit config.py.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker LXC (pull-at-boot) → /api/v1/internal/bootconfig/{vmid} | The one non-CRUD surface; a worker on the LAN fetches its own config + a short-lived credential |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-17 | Information Disclosure | bootconfig VMID enumeration / out-of-pool probe | mitigate | Validate `vmid ∈ [pool_start, pool_end]`; 404 WITHOUT echoing the probed value; int path param rejects non-int (ASVS V4/V5) |
| T-01-18 | Information Disclosure | git credential / Proxmox token in logs or event data | mitigate | Log only {vmid, repo}; credential never passed as a log `extra` or written to event `data`; JsonFormatter whitelists fields (ASVS V7, Pitfall 7, block_on=high gate) |
| T-01-19 | Spoofing | a non-worker caller requests another worker's config | mitigate | Source-IP defense-in-depth: `request.client.host == getIp(vmid)` (ADR-0004), gated by settings.bootconfig_source_ip_check; explicitly NOT auth (v1 LAN no-auth) |
| T-01-20 | Elevation of Privilege | long-lived / over-scoped git credential | mitigate | mint_repo_credential returns a short-lived, single-repo-scoped value (ADR-0002); no long-lived PAT hard-coded; issuer pluggable (A3); token read from gitignored .env only |
| T-01-21 | Information Disclosure | credential persisted to worker env | mitigate | Credential is fetched-and-discarded at boot, never written to /etc/burrow/worker.env (ADR-0002; the Phase-3 consumer enforces the discard) |
| T-01-SC | Tampering | package installs | mitigate | No new packages this plan; `uv lock --check` CI gate |
</threat_model>
