## Phase Goal

**As an** operator, **I want to** boot a worker that pulls its config + a short-lived credential from the control plane and clones my project without leaving the token behind, **so that** workspaces are reproducible and no credential is left on the worker.

<objective>
Replace the Phase-0 `burrow-boot.sh` stub with the live pull-at-boot spine: the worker self-resolves its VMID, bounded-retries the frozen `GET /api/v1/internal/bootconfig/{vmid}` endpoint, parses the camelCase `.data` envelope, clones the config + project repos using an in-memory `GIT_ASKPASS` one-shot credential (never in a URL, never persisted), and `exec`s the FROZEN persistent/LAN-bound ttyd. A hermetic Python `pytest` boot-harness (loopback fake control plane + `file://` bare git remotes + a stub `ttyd` on `PATH`) drives the script and proves the happy path plus the credential-no-leak guarantee — the worker-side analogue of the shipped `test_bootconfig.py` server-side gate.

This is the thin end-to-end slice: after this plan a worker boots, fetches, clones, and launches ttyd against a hermetic control plane, with the credential security core real (no stubbed auth). Manifest-driven plugins land in Plan 02; the ADR + CI wiring land in Plan 03.

Purpose: deliver WORK-02's live boot flow and the SC-3 credential-hygiene guarantee, CI-provable with zero real Proxmox.
Output: a live `burrow-boot.sh`, `jq` baked into the template, the `jsonschema` dev dep, and the `api/tests/boot/` harness.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker boot script → control-plane `/internal/bootconfig` | the worker pulls non-secret config + a short-lived credential over HTTP (operator-set `CONTROL_PLANE`) |
| short-lived credential → `git clone` | the minted token crosses into a subprocess; must never reach disk, a URL, an env file, or a log |
| `CONTROL_PLANE` env var → curl target | operator config in `worker.env` (trusted), never client input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | `clone_with_token` git credential | mitigate | GIT_ASKPASS one-shot in a subshell; `git -c credential.helper=` (empty); token NEVER in the clone URL (Pattern 1). Proven by `test_no_credential_leak`. |
| T-03-02 | Information Disclosure | `/etc/burrow/worker.env` | mitigate | pull-at-boot never writes a secret to the env file (ADR-0002); the scrub-proof test asserts the sentinel is absent from `worker.env` post-boot. |
| T-03-03 | Information Disclosure | ERR-trap / boot log line | mitigate | structural (token only in a subshell-local env var) + `$BASH_COMMAND` redaction backstop (Pattern 3) + no `set -x` (Pitfall 6). Mirrors the shipped `_safe()` + `test_bootconfig.py` no-leak gate. |
| T-03-04 | Denial of Service | `git clone` credential prompt (no TTY) | mitigate | `GIT_TERMINAL_PROMPT=0` → a bad token fails fast → ERR trap → fast non-zero (Pitfall 3), instead of hanging until the saga health-poll budget. |
| T-03-05 | Spoofing | VMID self-resolution → endpoint 404 | accept | the hostname→VMID parse is operator-recorded (ADR-0004 / 30-network-notes.md); a wrong VMID 404s safely. The parse is [ASSUMED] — confirm at the dev-homelab smoke. |
| T-03-SC | Tampering | apt install (`jq`) into the template | accept | `jq` is a distro-standard OS package (jqlang/jq, 10+ yrs) per the RESEARCH Package Legitimacy Audit — Approved; no PyPI/npm app dep is added here (jsonschema is a dev/CI dep, well-known, gated by lockfile freshness). |
</threat_model>
