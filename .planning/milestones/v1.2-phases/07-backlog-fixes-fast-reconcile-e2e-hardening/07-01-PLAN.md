# Phase 7 Plan 01 — Fast-Reconcile Wiring (UI-12) + E2E Hardening (CICD-09)

<objective>
Close the two v1.1 backlog findings. UI-12: wire the dead Pitfall-4 fast-reconcile so the
workspace list refreshes on a terminal error/close. CICD-09: harden stop-start.spec.ts with
panel-scoped locators + per-test backend isolation. Frontend/test only, CI-provable over the Fake.
</objective>

<threat_model>
Trust boundary: browser UI + e2e test harness only; LAN-only, no auth, no new network surface.
- (Integrity) The fast-reconcile invalidates the same-origin `WORKSPACES_KEY` query — no new
  endpoint, no status mirror into Zustand (server stays source of truth). Mitigation: assert no
  Zustand status write in the UI-12 test.
- (Test integrity) e2e cleanup must DELETE only workspaces this test created (id-scoped), never a
  broad wipe. Mitigation: track created ids per test, delete by id in afterEach.
No HIGH threats (frontend wiring + test refactor).
</threat_model>

<task id="2" name="CICD-09: harden stop-start.spec.ts (scoped locators + isolation)">
<read_first>
- ui/tests/e2e/stop-start.spec.ts (the target — unscoped .first(), global toHaveCount(0) ~line 80, no afterEach)
- ui/tests/e2e/terminal.spec.ts (cleanup / request patterns to reuse)
- ui/src/components/TerminalPanel.tsx (add a `data-testid={`panel-${id}`}` on the panel <section> if needed for scoping)
- ui/playwright.config.ts (do not modify; understand the harness)
</read_first>
<action>
Add a `data-testid="panel-${id}"` to the TerminalPanel root `<section>` (if no stable per-panel
hook exists). In stop-start.spec.ts, scope every interaction to the panel under test via a
`page.getByTestId(`panel-${...}`)` (or a `within`-style locator) — remove all unscoped `.first()`
and the global `[data-testid^="term-"].toHaveCount(0)`; assert instead that the under-test panel's
term body is absent and its `Workspace stopped` placeholder is shown. Add an `afterEach` that
deletes the workspace(s) the test created via a direct `request.delete('/api/v1/workspaces/{id}')`
(track ids as they are created), so the Fake backend does not accumulate and the suite is
order-independent. Remove the implicit single-panel dependency; keep `mode: serial` only if still
required.
</action>
<acceptance_criteria>
- No unscoped `.first()` and no global `[data-testid^="term-"]` count assertion remain in stop-start.spec.ts
- An `afterEach` destroys per-test created workspaces by id
- `cd ui && npm run build && npm run e2e` is green (the hardened spec passes, ports 8000/7681/4173 free first)
- `npm run typecheck` clean (the spec compiles)
