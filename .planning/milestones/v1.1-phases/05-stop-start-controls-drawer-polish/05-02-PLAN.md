<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API | `POST /api/v1/workspaces/{id}/stop|start` crosses here; the id is from the already-listed workspace, not free user text |
| UI render → backend state machine | the UI gates which action it offers; the backend `TRANSITIONS` table is the enforcement authority |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03 | Elevation (of action) | Stop/Start header buttons | mitigate | show-only-applicable gating (Stop iff running, Start iff stopped, neither otherwise) so the UI never offers an illegal lifecycle action; the backend `lib/statemachine.py` TRANSITIONS table rejects any that slips through with an envelope error |
| T-05-04 | Spoofing (of true state) | status reconcile | mitigate | no Zustand status mirror, no optimistic flip; the ~3s `useWorkspaces` poll re-lists the real server status after `onSettled` invalidation, so a backend rejection self-corrects rather than wedging |
| T-05-05 | (consistency) double-fire | stop/start mutation | mitigate | the `disabled` attribute while `isPending` is the double-fire guard; both Start affordances (header + placeholder CTA) share `startPending` and disable together (Pitfall 7) |
| T-05-06 | Tampering | new glyphs / spinner (no CDN) | mitigate | Stop/Start glyphs are inline outline SVG (no icon font, no CDN); the `googleapis\|gstatic\|jsdelivr` source assert stays green |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | no package install (no new runtime dependency — UI-SPEC Registry Safety, RESEARCH "Installation: none"); legitimacy gate n/a |

**Block-on-HIGH:** no HIGH threats by design — LAN-only, no-auth v1 frontend. No auth/session/crypto control is added (out of scope, CLAUDE.md). V7 error handling: a rejection surfaces as a readable envelope error + `console.error` (mirrors the existing destroy `onError`); the server-side redaction is already shipped, no secret reaches the client log.
</threat_model>
